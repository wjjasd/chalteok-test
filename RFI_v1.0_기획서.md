## 1. 서비스 개요

### 1.1 서비스명
**관계 적합도 인터뷰 — Relationship Fit Interview (RFI)**

### 1.2 목적
연애·파트너십 맥락에서 두 사람의 관계 적합도를 구조화된 자기 보고 설문을 통해 정량적으로 시각화하고, 사용자가 관계를 성찰할 수 있는 도구를 제공한다.

### 1.3 핵심 가치
본 서비스는 "진단"이 아닌 **"성찰 보조 도구"**로 포지셔닝한다. 점수는 관계의 절대적 결론이 아니라 대화와 자기 이해를 촉진하기 위한 참고 지표이다.

## 2. 법적 고지 및 면책 조항

### 2.1 서비스 이용 전 필수 동의 사항 (약관 동의 화면)

서비스 첫 진입 시 아래 내용을 **전문 표시 + 체크박스 동의** 방식으로 제공한다.

> **[면책 고지]**
>
> 1. 본 서비스는 심리 검사, 의료 진단, 상담 행위가 **아닙니다**. 결과는 학술적으로 검증된 표준화 심리 척도가 아니며, 어떠한 임상적·법적 효력도 갖지 않습니다.
> 2. 결과를 근거로 관계의 지속·종료·변경 등 중요한 결정을 내리는 것은 **권장하지 않습니다**. 중요한 관계 문제에 대해서는 반드시 전문 상담사 또는 심리 전문가의 도움을 받으시기 바랍니다.
> 3. 본 점수 체계는 완전하지 않으며, 인간 관계의 복잡성을 하나의 숫자로 환원할 수 없습니다. 결과를 맹신하지 마십시오.
> 4. 서비스 제공자는 본 서비스 이용으로 인해 발생하는 직·간접적 손해에 대해 법적 책임을 지지 않습니다.
> 5. 결과 공유 시 타인의 명예·프라이버시를 침해하지 않도록 유의하십시오. 결과를 이용하여 특정인을 비하·차별·공격하는 행위는 관련 법령에 따라 이용자 본인에게 책임이 있습니다.

### 2.2 개인정보 처리 원칙

| 원칙 | 내용 |
|------|------|
| 최소 수집 | 이름, 연락처, 주소, 주민등록번호 등 **개인정보보호법상 개인정보에 해당하는 항목은 일체 수집하지 않음** |
| 비식별성 | 수집 항목은 통계적 성격의 비식별 정보(연령대, 성별, 관계 단계 등)에 한정 |
| 서버 미저장 | 응답 데이터는 클라이언트(브라우저) 세션에서만 처리하며, 서버로 전송·저장하지 않음 |
| 쿠키·트래킹 | 분석을 위한 개인 식별 쿠키를 사용하지 않음 |

## 3. 화면 흐름 (User Flow)

```
[랜딩] → [약관 동의] → [사용자 프로필 입력] → [섹션 가중치 설정] → [섹션별 질문 응답 (A~H)] → [컷오프 항목 체크] → [결과 화면 (공유 기능)]
```

총 **10단계** 화면으로 구성한다.

---

## 4. 화면별 상세 설계

### 4.1 화면 ① — 랜딩 페이지

서비스 소개, CTA 버튼("시작하기"), 간략한 면책 요약 문구 표시.

### 4.2 화면 ② — 약관 동의

2.1절의 면책 고지 전문을 표시하고, 아래 두 항목에 모두 동의해야 진행 가능:

- "위 면책 고지를 읽었으며 동의합니다."
- "본 결과를 절대적 기준으로 사용하지 않겠습니다."

### 4.3 화면 ③ — 사용자 프로필 입력

설문 맥락 보정 및 향후 분석 고도화를 위한 비식별 정보를 수집한다. 현재 점수 산출에 직접 반영하지 않는 항목도 포함하되, 향후 가중치 보정·집단 통계 분석 등에 활용할 가능성을 열어둔다.

#### 수집 항목

| # | 항목 | 입력 형식 | 필수 여부 | 수집 목적 |
|---|------|-----------|-----------|-----------|
| P1 | 연령대 | 선택 (20대 초·중·후반, 30대 초·중·후반, 40대 이상) | 필수 | 연령대별 가중치 보정 가능성 |
| P2 | 성별 | 선택 (남성 / 여성 / 논바이너리 / 응답하지 않음) | 선택 | 통계 분류 |
| P3 | 현재 관계 단계 | 선택 (소개팅·첫만남 / 1~3개월 / 3~6개월 / 6개월~1년 / 1년 이상 / 동거·약혼·결혼) | 필수 | 섹션 활성화 조건, 문항 적용 범위 결정 |
| P4 | 과거 연애 횟수 | 선택 (0회 / 1~2회 / 3~5회 / 6회 이상) | 선택 | 경험치 맥락 |
| P5 | 관계에서 가장 중요하게 여기는 것 | 복수 선택 (신뢰 / 소통 / 정서적 안정 / 가치관 일치 / 생활 습관 / 신체적 끌림) — 최대 3개 | 필수 | 가중치 자동 추천 참고 |
| P6 | 현재 정서 상태 | 선택 (안정적 / 보통 / 다소 불안정 / 매우 불안정) | 선택 | 결과 해석 시 맥락 제공 |
| P7 | 평가 대상과의 갈등 빈도 | 선택 (거의 없음 / 월 1~2회 / 주 1~2회 / 거의 매일) | 선택 | Section C, G 해석 보정 |

> **참고:** 위 항목 중 어떤 것도 특정 개인을 식별할 수 없으며, 응답은 브라우저 세션에서만 유지된다.

### 4.4 화면 ④ — 섹션 가중치 설정

사용자가 8개 섹션의 상대적 중요도를 직접 설정한다. 이 가중치는 최종 100점 산출 시 각 섹션 점수에 곱해지는 비율을 결정한다.

#### 가중치 설정 방법: **100포인트 배분법 (Constant-Sum Method)**

사용자에게 총 **100포인트**를 제공하고, 8개 섹션에 자유롭게 배분하도록 한다.

| 섹션 | 기본값 (균등) | 사용자 배분 |
|------|---------------|-------------|
| A. 초기 끌림 | 12.5 | _____ |
| B. 대화 적합도 | 12.5 | _____ |
| C. 감정적 안정성 | 12.5 | _____ |
| D. 가치관 | 12.5 | _____ |
| E. 생활 적합도 | 12.5 | _____ |
| F. 신뢰 | 12.5 | _____ |
| G. 관계 동역학 | 12.5 | _____ |
| H. 자기 상태 변화 | 12.5 | _____ |
| **합계** | **100** | **100** |

#### UX 구현 방식

- **슬라이더 + 잔여 포인트 표시:** 각 섹션별 슬라이더(0~40 범위)를 제공하고, 상단에 "남은 포인트: XX"를 실시간 표시한다.
- **최소값 제약:** 각 섹션 최소 5포인트 이상 배분해야 한다. (특정 영역을 0으로 무시하는 것을 방지)
- **최대값 제약:** 단일 섹션에 40포인트 초과 불가. (하나의 섹션에 과도 편중 방지)
- **자동 추천 기능:** 화면 ③에서 선택한 P5("관계에서 가장 중요하게 여기는 것")를 기반으로 가중치를 자동 추천하는 버튼을 제공한다.
  - 예: 사용자가 "신뢰, 소통, 정서적 안정"을 선택했을 경우 → F(신뢰) 20, B(대화) 18, C(감정적 안정성) 18, 나머지 균등 배분
- **기본값 리셋 버튼:** 균등 배분(12.5)으로 되돌리는 버튼 제공.

### 4.5 화면 ⑤~⑫ — 섹션별 질문 응답 (Section A ~ H)

각 섹션은 독립 화면으로 구성한다. 한 화면에 해당 섹션의 5문항을 모두 표시한다.

#### 응답 UI

- 각 문항 아래 **5점 리커트 라디오 버튼** (0~4) 배치
- 역문항(Q5, Q10, Q15, Q20, Q25, Q30, Q35, Q40)은 UI상 동일하게 표시하되, **내부적으로 역채점(4→0, 3→1, 2→2, 1→3, 0→4)** 처리
- 문항 번호 옆에 역문항 표시는 하지 않음 (응답 편향 방지)
- 진행률 프로그레스바 상단 고정 표시

#### 관계 단계별 문항 활성화

화면 ③의 P3(관계 단계)에 따라 일부 섹션을 비활성화하거나 "해당 없음" 처리:

| 관계 단계 | 활성 섹션 | 비활성/선택 섹션 |
|-----------|-----------|-----------------|
| 소개팅·첫만남 | A, B | C~H (선택적 응답 가능, 미응답 시 해당 섹션 제외하고 점수 산출) |
| 1~3개월 | A, B, C, D, F | E, G, H (선택적) |
| 3개월 이상 | A~H 전체 활성 | — |

### 4.6 화면 ⑬ — 컷오프 항목 체크 (Critical Flags)

Q41~Q45에 대해 **YES / NO** 토글 버튼으로 응답.

이 항목들은 점수 체계와 **독립적으로** 작동하며, 결과 화면에서 별도의 경고 배지로 표시된다.

| YES 개수 | 판정 | 결과 화면 표시 |
|----------|------|---------------|
| 0 | 통과 | 표시 없음 |
| 1 | ⚠️ 주의 | 노란색 경고 배지 |
| 2 | 🔴 위험 | 빨간색 경고 배지 |
| 3 이상 | 🚨 강한 재검토 | 빨간색 전체 배너 경고 |

## 5. 점수 산출 체계

### 5.1 섹션 원점수 산출

각 섹션(5문항)의 원점수는 다음과 같이 계산한다:

```
섹션_원점수 = Σ(각 문항 점수)   (역문항은 역채점 후 합산)
섹션_만점 = 20 (5문항 × 4점)
섹션_백분율 = (섹션_원점수 / 20) × 100
```

### 5.2 가중 최종 점수 산출 (100점 만점)

사용자가 배분한 가중치(W)를 활용하여 최종 점수를 산출한다:

```
최종_점수 = Σ (섹션_백분율_i × W_i / 100)     (i = A~H)
```

여기서 W_i는 사용자가 배분한 포인트(합계 100)이다.

#### 산출 예시

| 섹션 | 원점수 (/ 20) | 백분율 | 사용자 가중치 W | 기여 점수 |
|------|--------------|--------|----------------|-----------|
| A. 초기 끌림 | 16 | 80% | 10 | 8.0 |
| B. 대화 적합도 | 18 | 90% | 15 | 13.5 |
| C. 감정적 안정성 | 14 | 70% | 15 | 10.5 |
| D. 가치관 | 12 | 60% | 20 | 12.0 |
| E. 생활 적합도 | 16 | 80% | 10 | 8.0 |
| F. 신뢰 | 18 | 90% | 15 | 13.5 |
| G. 관계 동역학 | 10 | 50% | 10 | 5.0 |
| H. 자기 상태 변화 | 17 | 85% | 5 | 4.25 |
| **합계** | | | **100** | **74.75** |

### 5.3 미응답 섹션 처리

관계 단계에 따라 비활성화된 섹션이 있을 경우:

```
최종_점수 = Σ (활성 섹션_백분율_i × W_i) / Σ (활성 섹션_W_i) × 100
```

→ 활성 섹션의 가중치 합으로 재정규화하여 100점 만점을 유지한다.

### 5.4 결과 등급 체계

| 점수 구간 | 등급 | 라벨 | 색상 |
|-----------|------|------|------|
| 85 ~ 100 | S | 높은 적합도 | 🟢 Green |
| 70 ~ 84 | A | 양호한 적합도 | 🔵 Blue |
| 55 ~ 69 | B | 보통 적합도 | 🟡 Yellow |
| 40 ~ 54 | C | 낮은 적합도 | 🟠 Orange |
| 0 ~ 39 | D | 부적합 신호 | 🔴 Red |

> **주의:** 등급은 관계의 성공/실패를 예측하는 것이 아니라, 현재 시점에서 자기 인식의 스냅샷을 제공하는 것이다.

## 6. 결과 화면 설계

### 6.1 구성 요소

#### (1) 총점 게이지
0~100 스펙트럼 게이지 바. 좌측(빨강, 부적합) → 우측(초록, 높은 적합도). 바늘이 최종 점수 위치를 가리킨다.

#### (2) 등급 배지
5.4절의 등급 라벨과 색상 표시.

#### (3) 섹션별 레이더 차트
8개 섹션의 백분율을 방사형(레이더/스파이더) 차트로 시각화. 강점·약점 영역이 직관적으로 보이도록 한다.

#### (4) 섹션별 점수 바
각 섹션의 백분율을 수평 바 차트로 표시. 가중치가 높은 섹션을 상단에 배치.

#### (5) 컷오프 경고 영역
4.6절의 판정 결과를 배지·배너로 표시. YES 항목이 있을 경우 해당 문항 내용을 함께 안내.

#### (6) 해석 가이드
점수 구간별 1~2문장의 해석 문구를 제공한다. 단, 아래 문구를 항상 함께 표시:

> "이 결과는 전문적인 관계 진단이 아닌, 자기 성찰을 위한 참고 자료입니다. 관계에 대한 중요한 결정은 신뢰할 수 있는 주변인 또는 전문 상담사와 상의하시기 바랍니다."

### 6.2 공유 기능

| 방법 | 구현 | 공유 내용 |
|------|------|-----------|
| URL 공유 | 결과 데이터를 URL 쿼리 파라미터 또는 Fragment(#)로 인코딩. 서버 저장 없이 클라이언트에서 복원 | 총점, 등급, 레이더 차트 |
| 이미지 저장 | html2canvas 등으로 결과 카드를 PNG/JPG 생성 후 다운로드 | 결과 요약 카드 이미지 |
| 카카오톡/SNS 공유 | 카카오 공유 SDK, Web Share API 활용 | OG 이미지 + 점수 요약 텍스트 |

#### 공유 시 주의 문구

공유 URL/이미지에 반드시 포함:

> "RFI는 자기 성찰 도구이며, 관계 진단이 아닙니다."

## 8. 향후 확장 고려 사항

### 8.1 프로필 데이터 활용 (v2.0)
현재 수집하는 P1~P7 데이터를 대규모 집단 통계(익명·비식별)로 축적하여, 관계 단계별·연령대별 가중치 추천 모델을 고도화할 수 있다.

### 8.2 커플 교차 비교 (v2.0)
두 사람이 각각 응답한 결과를 비교하여 인식 차이(Perception Gap)를 시각화하는 기능.

### 8.3 시계열 추적 (v2.0)
동일 사용자가 시간 간격을 두고 반복 측정하여 관계 변화 추이를 그래프로 제공.

### 8.4 전문가 연계 (v3.0)
컷오프 항목에서 위험 판정 시, 전문 상담 기관 안내 페이지로 연결하는 기능.

### 부록 C — 가중치 자동 추천 매핑표

사용자가 P5에서 선택한 항목을 기반으로 가중치를 자동 추천한다. 선택된 항목의 관련 섹션에 보너스 포인트를 배분하고, 나머지를 균등 배분한다.

| P5 선택 항목 | 관련 섹션 | 보너스 |
|-------------|-----------|--------|
| 신뢰 | F (신뢰) | +8 |
| 소통 | B (대화) | +8 |
| 정서적 안정 | C (감정적 안정성), H (자기 상태) | 각 +5 |
| 가치관 일치 | D (가치관) | +8 |
| 생활 습관 | E (생활 적합도) | +8 |
| 신체적 끌림 | A (초기 끌림) | +8 |

보너스 배분 후 합계가 100이 되도록 나머지 섹션에서 균등 차감한다.
